# Case Study: Implementation of SDLC Phases in a Real-World Engineering Project

## Project: Smart Parking Management System for a Metropolitan City

---

## 1. Introduction

The Smart Parking Management System (SPMS) was developed to address rising congestion in a metropolitan area. The system uses IoT sensors, mobile apps, and cloud-based data processing to help drivers locate available parking spaces in real time.

To ensure reliability and efficiency, the engineering team followed the Software Development Life Cycle (SDLC). This case study evaluates how each SDLC phase contributed to the projectâ€™s success.

---

## 2. Requirement Gathering

### Activities

- Conducted interviews with city authorities, traffic police, and citizens.
- Identified functional requirements (real-time parking availability, sensor data collection, mobile notifications).
- Defined non-functional requirements (high uptime, fast response time, secure data handling).

### Contribution to Outcomes

- Created a clear project scope, reducing future rework.
- Ensured alignment with city infrastructure and user needs.
- Established measurable performance benchmarks for later phases.

---

## 3. System Design

### Activities

- Designed system architecture integrating IoT sensors, cloud server, and mobile apps.
- Created ER diagrams, sequence diagrams, and API specifications.
- Selected technologies (AWS Cloud, MQTT protocol, React Native for mobile app).

### Contribution to Outcomes

- Improved system reliability through modular architecture.
- Ensured scalability to handle thousands of parking spots.
- Reduced integration issues during implementation due to well-defined interfaces.

---

## 4. Implementation

### Activities

- Developed backend microservices for sensor data processing.
- Built the mobile application for driver interactions.
- Implemented real-time dashboards for city authorities.

### Contribution to Outcomes

- Delivered a functional system that met all major design specifications.
- Modular coding allowed parallel development and faster delivery.
- Git-based CI/CD pipelines improved code quality and collaboration.

---

## 5. Testing

### Activities

- Conducted unit, integration, load, and security testing.
- Simulated peak traffic scenarios to validate performance.
- Field-tested sensors and network connectivity across multiple parking sites.

### Contribution to Outcomes

- Identified critical bottlenecks before deployment.
- Ensured high accuracy of real-time data.
- Increased system stability and reduced maintenance effort post-launch.

---

## 6. Deployment

### Activities

- Deployed backend services to AWS using auto-scaling groups.
- Released the mobile app on the Play Store and App Store.
- Trained city staff on system usage.

### Contribution to Outcomes

- Enabled a smooth, zero-downtime rollout.
- Ensured system availability from day one.
- Provided immediate usability to thousands of drivers.

---

## 7. Maintenance

### Activities

- Ongoing monitoring of sensor accuracy and server load.
- Monthly patch updates for performance improvements.
- Added new features, such as predictive parking availability using ML.

### Contribution to Outcomes

- Increased user satisfaction through continuous improvements.
- Enhanced system efficiency and reduced operational issues.
- Extended system lifespan and adaptability to changing city needs.

---

## 8. Conclusion

The Smart Parking Management System successfully transformed urban parking management by applying SDLC phases effectively.

Each SDLC phase contributed significantly:

- Requirement Gathering ensured clarity and alignment with real-world needs.
- Design created a scalable, robust architecture.
- Implementation turned the design into a functional, high-performance product.
- Testing ensured reliability and user trust.
- Deployment delivered the solution efficiently.
- Maintenance kept the system updated and future-ready.

The structured SDLC approach resulted in a well-planned, high-quality engineering solution that improved traffic flow, reduced congestion, and enhanced the overall user experience.
